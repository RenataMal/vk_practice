# VK Practice - Browser Image Enhancer

Браузерное приложение для автоматического улучшения изображений с помощью компактной ML-модели.

Модель анализирует изображение и подбирает коэффициенты:

- яркости;
- контрастности;
- насыщенности.

После инференса параметры применяются к полноразмерному изображению внутри Web Worker. Обработка выполняется локально, без отправки пользовательских файлов на сервер и без блокировки основного потока интерфейса.

## Поддерживаемые форматы

| Входной формат | Выходной формат |
|---|---|
| JPG / JPEG | JPG |
| PNG | PNG |
| BMP | PNG |
| HEIC / HEIF | JPG |

Для BMP поддерживаются несжатые 24-битные и 32-битные изображения.

## Архитектура

```text
Пользователь выбирает изображение
                ↓
ImageEnhancer API создаёт задачу
                ↓
Задача помещается в очередь
                ↓
Web Worker декодирует изображение
                ↓
Изображение преобразуется в тензор 3 × 96 × 96
                ↓
Компактная CNN предсказывает 3 коэффициента
                ↓
К параметрам применяются защитные ограничения
                ↓
Canvas 2D применяет коррекцию к оригиналу
                ↓
Результат кодируется в JPG или PNG
                ↓
Пользователь скачивает готовое изображение
```

## JavaScript API

Основной класс:

```typescript
const enhancer = new ImageEnhancer();
```

Постановка задачи:

```typescript
const taskId = await enhancer.submit(file);
```

Получение статуса:

```typescript
const status = await enhancer.getStatus(taskId);
```

Получение результата:

```typescript
const result = await enhancer.getResult(taskId);
```

Отмена задачи:

```typescript
await enhancer.cancel(taskId);
```

Подписка на изменение статуса:

```typescript
enhancer.addEventListener(
  'statuschange',
  (event) => {
    console.log(event.detail);
  },
);
```

Статусы задачи:

```text
queued
decoding
analyzing
enhancing
encoding
completed
cancelled
failed
```

## ML-модель

Модель получает RGB-изображение размером `96 × 96` и возвращает три значения:

```text
brightness
contrast
saturation
```

Архитектура:

```text
Conv2D 3 → 16
ReLU
Conv2D 16 → 32
ReLU
Conv2D 32 → 48
ReLU
Conv2D 48 → 64
ReLU
Global Average Pooling
Linear 64 → 32
ReLU
Linear 32 → 3
Sigmoid
```

### Обучающие данные

Финальная модель обучалась на качественных изображениях из LOL Dataset.

В качестве эталонных использовались изображения из папок `high`. Для каждого изображения программно создавались контролируемые искажения яркости, контрастности и насыщенности. Модель обучалась восстанавливать известные параметры коррекции и исходное изображение.

Разбиение данных:

| Выборка | Исходных изображений | Сгенерированных примеров |
|---|---:|---:|
| Train | 412 | 3296 за эпоху |
| Validation | 73 | 438 |
| Test | 15 | 180 |

Использованные диапазоны синтетических параметров:

| Параметр | Минимум | Максимум |
|---|---:|---:|
| Яркость | 0.65 | 1.65 |
| Контрастность | 0.75 | 1.35 |
| Насыщенность | 0.70 | 1.40 |

### Результаты модели

Лучшая модель была сохранена на 15-й эпохе.

| Метрика | Значение |
|---|---:|
| Test loss | 0.13370 |
| Test parameter MAE | 0.08641 |
| Диапазон прогнозов яркости | 0.8852 |
| Диапазон прогнозов контрастности | 0.2722 |
| Диапазон прогнозов насыщенности | 0.0539 |

Стандартное отклонение прогнозов на тестовой выборке:

```text
brightness: 0.1613
contrast:   0.0578
saturation: 0.0101
```

Диагностика на отдельных тёмных, светлых и обычных фотографиях показала, что модель выдаёт разные параметры в зависимости от входного изображения и не схлопывается к одному постоянному прогнозу.

### Защитные ограничения

Перед применением к изображению параметры ограничиваются безопасными диапазонами:

| Параметр | Минимум | Максимум |
|---|---:|---:|
| Яркость | 0.82 | 1.55 |
| Контрастность | 0.85 | 1.35 |
| Насыщенность | 0.85 | 1.35 |

При сильном осветлении или затемнении контрастность не опускается ниже `0.90`. Это уменьшает риск серой пелены на тёмных фотографиях и чрезмерного затемнения светлых изображений.

## Компактный runtime

Стандартный ONNX Runtime Web увеличивал размер браузерной сборки примерно на 13 МБ и нарушал ограничение проекта.

Поэтому для инференса реализован собственный TypeScript runtime, выполняющий только операции, используемые моделью:

- Conv2D;
- ReLU;
- Global Average Pooling;
- Linear;
- Sigmoid.

Конфигурация и веса модели хранятся в файлах:

```text
app/public/models/model-config.json
app/public/models/model-weights.bin
```

## Размер решения

Размер production-сборки с ML-весами и HEIC-декодером составляет около `3.1 МБ`.

Требование:

```text
не более 10 МБ
```

Требование выполнено.

## Performance benchmark

Финальный benchmark выполнялся в браузере Microsoft Edge на синтетических изображениях размером:

- 1 Мп;
- 5 Мп;
- 10 Мп;
- 15 Мп.

Каждый размер обрабатывался три раза.

Итоговые результаты:

| Метрика | Значение |
|---|---:|
| Количество запусков | 12 |
| Среднее время обработки | 413,8 мс |
| Максимальное внутреннее время | 706,9 мс |
| Максимальный wall time | 707,3 мс |
| Максимальный размер изображения | 15 Мп |
| Ошибки обработки | 0 |

Проверка требований:

| Требование | Результат |
|---|---|
| Среднее время не более 5 секунд | выполнено |
| Максимальное время не более 30 секунд | выполнено |
| Обработка изображений до 15 Мп | выполнено |

На больших изображениях основную часть времени занимает кодирование результата, а не ML-инференс. Фактическая скорость зависит от устройства, браузера, формата и сложности изображения.

```bash
cd app
npm install
npm run dev
```

Приложение будет доступно по адресу, указанному Vite.

```bash
npm run build
```

## Обучение модели

Активировать Python-окружение:

```bash
source .venv/Scripts/activate
```

Установить зависимости:

```bash
python -m pip install -r ml/requirements.txt
```

Датасет LOL должен находиться в структуре:

```text
ml/data/lol/
├── our485/
│   ├── low/
│   └── high/
└── eval15/
    ├── low/
    └── high/
```

Финальное обучение:

```bash
python -m ml.src.train_lol_balanced_synthetic \
  --epochs 20 \
  --batch-size 16 \
  --learning-rate 0.0005
```

Диагностика модели:

```bash
python -m ml.src.diagnose_balanced_synthetic_model
```

Подготовка основной модели и экспорт браузерных весов:

```bash
cp ml/artifacts/models/best_balanced_synthetic_model.pt \
   ml/artifacts/models/best_model.pt

python -m ml.src.export_browser_weights
```

После экспорта необходимо пересобрать приложение:

```bash
cd app
npm run build
```

## Ограничения

- BMP поддерживается только без сжатия с глубиной 24 или 32 бита.
- HEIC и HEIF преобразуются в JPG.
- Обработка изображений более 15 Мп блокируется.
- Модель выполняет глобальную коррекцию и не изменяет отдельные области изображения независимо.
- Защитные ограничения могут приводить к одинаковым итоговым параметрам для нескольких сильно тёмных или сильно светлых фотографий.
- Насыщенность распознаётся слабее яркости и контрастности.
- Производительность зависит от устройства и браузера.
- Основная функциональная проверка выполнялась в Microsoft Edge; полноценная кросс-браузерная проверка требует отдельных тестов в Chrome, Firefox и Safari.
- Качество результата зависит от распределения изображений, использованных при обучении.
